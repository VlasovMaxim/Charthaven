ОПИСАНИЕ ИНТЕРФЕЙСА И ФУНКЦИОНАЛА ВЕБ-ПРИЛОЖЕНИЯ CHARTHAVEN
ОБЩАЯ КОМПОВКА ГЛАВНОЙ СТРАНИЦЫ
Главная страница веб-приложения условно делится на две части. Слева расположена
боковая панель (сайдбар): в ней сосредоточен основной сценарий работы —
загрузка файла, настройка графика, список сохранённых графиков, справочный
раздел «О приложении». На широком экране панель занимает заметную, но не
половину ширины (в вёрстке ориентир порядка 30 % рабочей области), справа
остаётся основное пространство под график. Справа располагается область
отображения графика: заголовок текущего графика, элементы экспорта и сама
диаграмма, строимая библиотекой Plotly в браузере.
В самом верху страницы расположена шапка сайта. Слева в шапке — логотип
и ссылка на главную; для узких экранов на главной доступна кнопка «бургер»,
открывающая боковую панель поверх контента. В шапке также есть переключатель
светлой и тёмной темы оформления. Справа в шапке для авторизованного
пользователя отображается кнопка с аватаром (или заглушкой) и именем, по нажатию
открывается меню: переход в личный кабинет и выход. Если пользователь не вошёл
в систему, справа показываются ссылки «Войти» и «Регистрация».
1) ОБЛАСТЬ ОТОБРАЖЕНИЯ ГРАФИКА (СПРАВА)
Эта часть интерфейса логически разделена на два уровня.
Сверху находится панель инструментов графика. Слева на ней выводится название
текущего графика (заголовок, заданный пользователем в настройках, либо пусто,
если название не задавали). Справа расположен блок скачивания: выпадающий список
формата файла (PNG, SVG или JPEG) и кнопка загрузки графика на устройство
пользователя. Экспорт учитывает выбранные параметры оформления и тему интерфейса
там, где это задано в логике приложения.
Ниже располагается крупная область (контейнер), в которой Plotly отрисовывает
график или диаграмму. Пока данные не загружены или график не построен, область
может оставаться пустой; после построения пользователь может масштабировать
и просматривать диаграмму стандартными средствами Plotly.
2) БОКОВАЯ ПАНЕЛЬ (СЛЕВА)
Через боковую панель выполняется основная работа с приложением. Она состоит
из двух уровней по вертикали.
В верхней части панели расположены вкладки переключения режимов:
  • «Загрузка данных»
  • «Настройки графика»
  • «Созданные графики» (в интерфейсе также используется формулировка про
    сохранённые графики)
  • «О приложении»
До загрузки файла вкладка «Настройки графика» недоступна (заблокирована); после
успешной загрузки она становится активной. Вкладка «Созданные графики» для
неавторизованного посетителя недоступна: сохранение коллекции графиков
привязано к учётной записи.
В нижней части панели отображается содержимое выбранной вкладки — формы,
списки, подсказки.

ФУНКЦИОНАЛ ПО ВКЛАДКАМ
1) Вкладка «Загрузка данных»
Здесь пользователь загружает табличный файл в формате CSV или Excel (.xlsx, .xls).
Предполагается, что таблица уже приведена к виду, удобному для построения
графиков: есть именованные столбцы с данными, пригодными для выбора в качестве
осей или категорий. Приложение не выполняет глубокой аналитики или автоматического
«очищения» данных за пользователя: оно читает файл, показывает краткий
предпросмотр первых строк и передаёт список имён столбцов в настройки графика.
Дополнительно поддерживается перетаскивание файла в зону выбора (drag-and-drop).
После загрузки можно удалить привязанный файл кнопкой и загрузить другой.
2) Вкладка «Настройки графика»
После загрузки файла пользователь переходит на эту вкладку. Сначала выбирается
тип графика. Типы представлены в виде карточек с пиктограммами: линейный,
столбчатый, круговой, гистограмма, точечный, с областями. В зависимости от
выбранного типа отображается свой набор полей: оси X и Y (где применимо),
подписи осей, заголовок, цвет (для круговой диаграммы — набор цветов сегментов),
для линейного графика — толщина и стиль линии, для гистограммы — число корзин
и т.д.
Имеется блок «расширенных» настроек оформления: шрифт, цвет подписей, цвет фона
графика, отображение сетки, легенда и её положение. Подсказка в интерфейсе
указывает, что график перестраивается автоматически при выборе типа и при
изменении настроек (отдельная кнопка «Создать график» для первичного построения
не требуется — отличие от упрощённого сценария, где график создаётся одной кнопкой).
Чтобы добавить текущий график в коллекцию на сервере, используется кнопка
«Сохранить график». Она доступна смыслово только для вошедшего пользователя:
гость может строить и смотреть график, но не сохраняет его в список «Созданные
графики». При сохранении на сервер передаются параметры построения и данные
графика; дубликаты по нормализованному названию графика отклоняются.
3) Вкладка «Созданные графики»
Для авторизованного пользователя здесь отображается список ранее сохранённых
графиков. Элементы списка показывают информацию о записи: в том числе имя
исходного файла, тип графика, дату и время создания, название графика (если
задавалось). Доступны поиск по строке, сортировка (по дате, имени файла,
названию, типу) и группировка (по файлу, типу, дню).
По нажатию на элемент списка график снова отображается в правой области;
одновременно на клиенте восстанавливаются связанные с ним настройки в формах
боковой панели (тип, оси, заголовок, цвета, расширенное оформление), панель
переключается на вкладку «Настройки графика», чтобы пользователь при желании
мог отредактировать параметры и пересохранить. Скачивание по-прежнему доступно
через панель над областью графика.
Удаление записи из коллекции предусмотрено в сценарии работы со списком
(через интерфейс сохранённых графиков).
4) Вкладка «О приложении»
Здесь размещена краткая информация о продукте ChartHaven: название, слоган,
текст о назначении, маркированный перечень возможностей (форматы файлов,
типы диаграмм, оформление, сохранение в аккаунте, экспорт, адаптивность),
ссылки-заготовки (например, на репозиторий и контакт по электронной почте —
в шаблоне они задаются как примеры и при необходимости заменяются на реальные).

РЕГИСТРАЦИЯ И АВТОРИЗАЦИЯ
Регистрация и вход реализованы для того, чтобы пользователь мог хранить
созданные графики в приложении и возвращаться к ним во вкладке «Созданные
графики». Без входа в систему доступны загрузка файла, предпросмотр,
построение и экспорт графика в рамках текущей сессии браузера.
Отдельно доступны страницы «Войти», «Регистрация» и «Личный кабинет». В личном
кабинете можно сменить логин, загрузить аватар и изменить пароль. Тема
оформления (светлая/тёмная) для авторизованного пользователя может сохраняться
в профиле на сервере; для гостя выбор темы хранится локально в браузере.

ХРАНЕНИЕ ДАННЫХ
Данные пользователей, загруженные файлы (в виде двоичных полей), параметры
и сериализованные графики сохраняются в базе данных через механизм ORM Django
(в проекте используется MySQL). То есть приложение опирается на полноценную
реляционную базу данных, а не только на сессии или файлы на диске без СУБД.
