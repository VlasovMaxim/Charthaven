# Charthaven

Веб-приложение для загрузки табличных данных (CSV, Excel), построения интерактивных графиков (Plotly) и сохранения настроек в браузере. Для зарегистрированных пользователей доступна коллекция сохранённых графиков в базе данных и личный кабинет.

Интерфейс на русском языке. Серверная часть — **Django**, визуализация — **Plotly.js** на клиенте и **Plotly Python** при генерации графика на сервере.

## Возможности

- Загрузка файлов **CSV** и **Excel** (`.csv`, `.xlsx`, `.xls`); данные файла хранятся в базе и привязываются к учётной записи или к сессии гостя.
- Типы графиков: линейный, столбчатый, круговой, гистограмма, точечный, с областями.
- Настройка осей, подписей, цветов, стиля линий, легенды, сетки, фона и шрифтов; часть параметров сохраняется в **localStorage** и переживает обновление страницы и смену типа графика.
- Светлая и тёмная тема интерфейса; для авторизованных пользователей тема может сохраняться в профиле.
- Регистрация, вход, выход, личный кабинет (логин, аватар, смена пароля).
- Для авторизованных пользователей: сохранение графика в коллекцию **«Созданные графики»**, просмотр списка, открытие и удаление записей.
- Экспорт изображения графика (форматы, доступные в интерфейсе скачивания).
- Адаптивная вёрстка (в том числе боковое меню на узких экранах).

## Гость и авторизованный пользователь

| Действие | Гость (не вошёл в аккаунт) | Авторизованный пользователь |
|----------|----------------------------|-----------------------------|
| Загрузка файла и построение графика | Да, привязка к **сессии Django** | Да, файл привязан к **пользователю** |
| Настройки графика в браузере (localStorage) | Да | Да |
| Сохранение графика в коллекцию «Созданные графики» | Нет (сервер возвращает ошибку при попытке сохранить в коллекцию) | Да |
| Вкладка «Созданные графики» | Открывается, но список и загрузка сохранённых требуют входа; показывается сообщение, вкладка сбрасывается на «Загрузка данных» | Полный доступ к списку и карточкам |
| Тема интерфейса в профиле на сервере | Нет | Да (API темы) |

| Слой | Технологии |

## Установка и запуск

1. Клонировать репозиторий и перейти в каталог проекта:

```bash
git clone git@github.com:VlasovMaxim/Charthaven.git
cd Charthaven
```

(Если каталог после клонирования называется иначе, перейдите в него вместо `Charthaven`.)

2. Создать и активировать виртуальное окружение:

```bash
python -m venv .venv
```

Linux и macOS:

```bash
source .venv/bin/activate
```

Windows (PowerShell):

```powershell
.venv\Scripts\Activate.ps1
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Настроить подключение к базе данных (см. следующий раздел).

5. Применить миграции:

```bash
python manage.py migrate
```

6. Запустить сервер разработки:

```bash
python manage.py runserver
```

По умолчанию приложение доступно по адресу: `http://127.0.0.1:8000/`.

Для доступа к админке Django создайте суперпользователя:

```bash
python manage.py createsuperuser
```

Админка: `http://127.0.0.1:8000/admin/`.

## База данных

В файле `chartHaven/settings.py` в секции `DATABASES` задаётся подключение к **MySQL**: имя базы, пользователь, пароль, хост и порт. Перед первым запуском создайте пустую базу в MySQL и укажите те же параметры в настройках.

Рекомендация для репозитория и продакшена: не коммитить реальные пароли; вынести секреты в переменные окружения или файл `.env` (в репозитории обычно хранят только пример `.env.example` без секретов). В `requirements.txt` указан `python-dotenv` для удобной загрузки переменных из `.env` при доработке настроек.

## Тесты

Автотесты приложения `chart` можно выполнять с отдельным модулем настроек **`chartHaven.settings_test`**, где база данных — **SQLite в памяти**. Так не требуется запущенный MySQL:

```bash
python manage.py test chart.tests --settings=chartHaven.settings_test
```

Обычный запуск `python manage.py test` использует базу из основного `settings.py` (у вас это MySQL).

## Сценарии использования

### Работа без регистрации

1. Откройте главную страницу.
2. На вкладке «Загрузка данных» выберите CSV или Excel и отправьте файл.
3. Перейдите на «Настройки графика», выберите тип, оси и остальные параметры; график обновляется через API `create_chart`.
4. Настройки и идентификатор загруженного файла хранятся в **localStorage**; для гостя файл в БД связан с **сессией браузера** (нужны cookies, в запросах используется передача CSRF).

Сохранение в коллекцию «Созданные графики» для гостя недоступно: при попытке сервер отвечает ошибкой с просьбой войти.

### Работа с аккаунтом

1. **Регистрация** или **Вход** через пункты в шапке.
2. Те же шаги загрузки и настройки графика.
3. Кнопка сохранения графика в коллекцию (если предусмотрена в интерфейсе) записывает график в БД и привязывает его к пользователю и файлу.
4. Вкладка **«Созданные графики»** загружает список через API; можно открыть сохранённый график или удалить его.
5. **Личный кабинет** — правка логина (с проверками), аватар, смена пароля; для авторизованных доступно сохранение темы интерфейса на сервере.

## Маршруты приложения

Основные пути (приложение `chart`, корень сайта):

| Путь | Назначение |
|------|------------|
| `/` | Главная страница с конструктором графиков |
| `/upload/` | POST: загрузка файла (multipart) |
| `/create_chart/` | POST: JSON с параметрами графика; ответ с данными Plotly |
| `/register/`, `/login/`, `/logout/` | Регистрация, вход, выход |
| `/profile/` | Личный кабинет |
| `/register/check-username/`, `/register/validate-password/` | AJAX-проверки при регистрации |
| `/profile/verify-password/`, `/profile/check-username/` | AJAX-проверки в профиле |
| `/api/user-theme/` | POST: сохранение темы (только авторизованные) |
| `/api/saved-charts/` | GET: список сохранённых графиков |
| `/api/saved-charts/<id>/` | GET: данные графика; DELETE: удаление |

Точные имена маршрутов см. в `chart/urls.py`.

## Фронтенд и статика

- Стили: `chart/static/chart/css/` (`reset.css`, `styles.css`, `auth.css`, `profile.css`, `theme.css` и др.).
- Скрипты главной страницы разнесены по файлам (порядок подключения в `base.html` важен):

  - `chart-core.js` — модальные окна, CSRF, `escapeHtml`, drag-and-drop файла;
  - `chart-workspace.js` — вкладки, загрузка, Plotly, настройки графика, localStorage;
  - `chart-saved.js` — коллекция «Созданные графики»;
  - `chart-init.js` — инициализация после загрузки DOM (восстановление вкладок, бургер-меню).

- В шаблонах для сброса кэша браузера у статики могут использоваться query-параметры вида `?v=…`. В `settings.py` закомментирован вариант с `ManifestStaticFilesStorage` для продакшена после `collectstatic`.

Статика в режиме разработки отдаётся через `django.contrib.staticfiles` при `DEBUG=True`; для продакшена выполняют `collectstatic` и настраивают веб-сервер на раздачу файлов из каталога статики.

## Безопасность и продакшен

Перед выкладкой в продакшен:

- Задайте уникальный `SECRET_KEY`, отключите `DEBUG`, укажите `ALLOWED_HOSTS`.
- Используйте HTTPS и корректные настройки cookie сессии.
- Не храните пароли БД и секреты в коде; используйте переменные окружения.
- Ограничьте размер загружаемых файлов и следите за нагрузкой при обработке больших таблиц (pandas в памяти).
